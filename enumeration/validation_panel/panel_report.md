# Route-3 enumeration — validation panel (frozen)

Companion to `panel_results.csv` and `logs/`. Generated by `../route3_panel.py`
(deterministic: box counts reproduce exactly across runs).

**Method.** Certified real-box enumeration of a 5-constraint subset by
branch-and-prune: tight *affine-arithmetic (AA)* exclusion discards root-free
boxes; an *AA-Krawczyk* interval-Newton test (Jacobian enclosed by forward-mode AD
in affine arithmetic) certifies unique real roots. A run is **complete**
(`done=True`) when every sub-box is either AA-excluded or Krawczyk-decided, so a
complete run is a *count*, not a sample.

**Rigor status.** Proof-of-principle (float-coefficient AA). Krawczyk contraction
margins are large (0.169 at r=3e-3, not 0.999), so floating-point rounding cannot
flip any verdict here. Bit-level certification = rounded AA / Taylor models (Julia),
deferred to the hardening step.

**Search box (illustrative — NOT the registered pre-registration box).**
b[0.20,0.80] c[0.10,0.45] d[0.15,0.45] e[0.25,0.75] g[0.03,0.25] (Rao range + 50%).

## Findings

- **The method generalizes** across reproduce / grounded / random / infeasible /
  dense / degenerate classes, and is soundness-checked against an independent solver.
- **Not every well-posed subset is feasible in the box** (coord-heavy, dense have no
  real figure). The campaign certifies *absence* as often as presence.
- **Degeneracy is self-announcing** three ways: queue inflation, no certification,
  and a Newton-detected continuum.

## Caveats (carry into Amendment-02)

1. Float-coefficient AA ⇒ proof-of-principle rigor; counts cannot flip given the
   margins, but bit-level certification needs rounded AA / Taylor models.
2. Certified roots located to ~box radius (3e-3); a Newton polish gives full precision.
3. Illustrative box, not the registered pre-registration box.
4. Rank-deficient subsets must be skipped at Gate 1.

## Execution-risk note (next milestone)

Correctness is demonstrated; **campaign feasibility is not**. Runtimes here range
~40–96 s (single core), but this panel was not chosen to characterise the runtime
*distribution*. Before any 815-subset launch: a 30–50 subset runtime-distribution
study (median / 90th-pct / worst-case boxes & time, feasible fraction) → only then a
defensible campaign estimate.
